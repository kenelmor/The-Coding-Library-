# Media Files and Screenshots

## Screenshots Instructions

To complete the project requirements, please take screenshots of the running application and save them in this folder.

### Recommended Screenshots:

1. **Main GUI Window** - The initial view with the book table showing all pre-loaded books
   - Filename suggestion: `01-main-window.png`

2. **Adding a Book** - Fill in the ISBN, Title, and Author fields and show the "Add Book" button
   - Filename suggestion: `02-add-book.png`

3. **Book Table with New Book** - After adding a new book, showing it appears in the table
   - Filename suggestion: `03-book-added.png`

4. **Checking Out a Book** - Select a book from the table and click "Check Out"
   - Filename suggestion: `04-check-out.png`

5. **Book Checked Out Status** - Show the book's status changed to "Checked Out" in the table
   - Filename suggestion: `05-checked-out-status.png`

6. **System Messages** - Show the message area at the bottom with logged actions
   - Filename suggestion: `06-system-messages.png`

7. **Check In Operation** - Returning a book back to the library
   - Filename suggestion: `07-check-in.png`

8. **Command Line Interface (Optional)** - Screenshot of LibraryManager running in terminal
   - Filename suggestion: `08-cli-interface.png`

### How to Take Screenshots:

**On macOS:**
- Press `Cmd + Shift + 4` and drag to select the window area
- Or press `Cmd + Shift + 4`, then press `Space` and click on the window

**On Windows:**
- Press `Windows + Shift + S` to use Snipping Tool
- Or use `Alt + PrtScn` to capture the active window

**On Linux:**
- Use the Screenshot tool (varies by distribution)
- Or use `gnome-screenshot` command

### After Taking Screenshots:

1. Save all screenshots to this `media/` folder
2. Name them descriptively (as suggested above)
3. Commit and push them to the GitHub repository:
   ```bash
   git add media/
   git commit -m "Add application screenshots"
   git push
   ```
